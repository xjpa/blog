## Building Low Latency Applications with C++

<img src="/photos/books/building-low-latency-applications-cpp.jpeg" height ="300">

I got this on kindle about a week from the book's release, here's a pic of it from my balcony in the morning. Kindle says it's 01:27 AM, but I havent set it correctly yet from a flight

<a href="/photos/books/page/building-low-latency-apps-cpp-kindle.jpeg"><img src="/photos/books/page/building-low-latency-apps-cpp-kindle.jpeg"></a>

As I was early, there was no trustworthy reviews about it yet but the title caught my eye. I have been learning some C++ at the time and I wanted to get into some project based learning.

Turns out, the book doesnt really get you into building the project from scratch. But it does get you introduced into some components needed for a trading application. Looking at the C++ snippets from the book, I've decided to build one with Golang as the scope of the book's C++ is beyond my C++ understanding.

But I do love how it gives caution to several things that would get myself shot in the foot, much of these are from modern C++. Such as it talks about using caution with `std::bind`. A lot of it is in line with what my software engineering mentor (currently working at Snapchat) is telling me.

Despite its fallings, I would recommend this book though for noobs like me
